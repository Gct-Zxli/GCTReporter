## 📝 文档编写规范

### 文档命名规范
- 使用中文命名，语义清晰
- 多个单词用短横线连接
- 添加版本号（如v1.0）
- 示例：`PROJECT_STATUS_v1.0.md`

### Markdown格式规范
- 使用标准Markdown语法
- 代码块指定语言类型
- 表格对齐格式化
- 链接使用相对路径

### 文档更新流程
1. 修改文档内容
2. 更新文档头部的"最后更新"日期
3. 在Git commit中说明修改内容
4. 重要文档变更需要Code Review
